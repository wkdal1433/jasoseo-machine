# 🎬 GIF 제작용 단일 연속 녹화 가이드

이 가이드는 **한 번의 연속 녹화**로 모든 GIF와 스크린샷을 커버하기 위한 촬영 스크립트입니다.
녹화 후 편집 툴에서 타임스탬프 기준으로 잘라내어 각 파일을 완성합니다.

---

## 촬영 전 준비

```
앱 창 크기: 1280×800 (고정)
녹화 툴: ScreenToGif (Windows) — 전체 녹화 모드
녹화 fps: 24fps
Chrome: 채용 사이트 탭 미리 열어두기 (자소서 입력 폼 있는 페이지)
앱 상태: 에피소드 카드 3개 이상 있는 프로필로 로그인된 상태
```

> **핵심 원칙**: 모든 동작을 **평소보다 30% 느리게** 합니다. 편집 시 빠르게 할 수 있지만 느리게 만들 수는 없습니다.

## 단계별 촬영 스크립트

### ▶ 녹화 시작

---

### [A구간] 0:00 ~ 0:10 — 대시보드 (`dashboard.png`)

1. 대시보드 화면이 보이는 상태에서 **5초 정지** (마우스 움직이지 않음)
2. 사이드바 + 지원서 카드 2~3개 보이게 스크롤 조정
3. 다시 **3초 정지**

> 📸 **이 구간에서 `dashboard.png` 스크린샷** (2~3초 지점)

---

### [B구간] 0:10 ~ 0:35 — 매직 온보딩 (`onboarding.gif`)

1. 사이드바 또는 대시보드에서 **[✨ 매직 온보딩]** 버튼 클릭 (천천히)
2. 온보딩 화면 열리면 **2초 정지**
3. PDF 파일을 드래그 드롭 — **드래그 궤적이 보이도록 천천히** 화면을 가로질러 이동
4. 파일이 올라가면 **1초 정지**
5. AI 분석 로딩 스피너 — **그대로 대기** (빨리 넘기지 않음)
6. 에피소드 카드들이 순차적으로 등장 — **카드 3~4개 전부 뜰 때까지 대기**
7. 마지막 카드 뜬 후 **2초 정지**

> 🎬 **B구간 전체 = `onboarding.gif`** (0:10 ~ 0:35 추출)

---

### [C구간] 0:35 ~ 0:50 — 에피소드 라이브러리 (`episodes.png`)

1. 에피소드 탭으로 이동 (카드 그리드 화면)
2. HR 의도 태그(실행력/성장성 등)와 역량 뱃지가 보이도록 스크롤
3. **5초 정지**

> 📸 **이 구간에서 `episodes.png` 스크린샷** (3~4초 지점)

---

### [D구간] 0:50 ~ 1:10 — 스마트 URL 모드 (`smart-mode.gif`)

1. **[새 지원서]** 버튼 클릭 (천천히)
2. URL 입력 모드 탭 선택 후 **1초 정지**
3. 채용 사이트 URL 붙여넣기 — **Ctrl+V 후 1초 정지** (URL이 보이도록)
4. **[공고 자동 분석]** 버튼 클릭
5. 로딩 애니메이션 — **그대로 대기**
6. 문항 목록 자동 생성 완료 화면 — **3초 정지**

> 🎬 **D구간 전체 = `smart-mode.gif`** (0:50 ~ 1:10 추출)

---

### [E구간] 1:10 ~ 1:20 — Step 0 기업 리서치 (`step0.png`)

1. 직무 선택 후 **[다음]** 또는 Step 0 분석 결과 화면으로 이동
2. 인재상 분석 카드 + HR 의도 분류 + 작성 전략이 보이는 상태
3. **5초 정지**

> 📸 **이 구간에서 `step0.png` 스크린샷** (2~3초 지점)

---

### [F구간] 1:20 ~ 1:40 — Step 1~2 브릿지 (hero.gif 연결용)

1. Step 1 질문 재해석 화면 — **2초 정지**
2. Step 2 에피소드 매칭 카드 선택 — **천천히** 카드 클릭
3. **[승인]** 버튼 클릭 — **1초 정지**

> 이 구간은 독립 GIF 없음. hero.gif의 중간 연결 장면으로만 사용.

---

### [G구간] 1:40 ~ 2:10 — 실시간 스트리밍 생성 (`generation.gif`)

1. **[생성 시작]** 버튼 클릭
2. 재밌는 멘트 로테이션 + shimmer 바 — **그대로 대기** (스킵 금지)
3. 첫 글자 등장 순간 — **이 장면부터가 핵심**
4. **글자가 촤르륵 채워지는 스트리밍 장면** — 끝까지 대기
   - 상단 진행 바 + 글자수 카운터가 화면에 보이는지 확인
5. 생성 완료 후 **3초 정지**

> 🎬 **G구간 전체 = `generation.gif`** (1:40 ~ 2:10 추출)

---

### [H구간] 2:10 ~ 2:20 — Full Review 완성 화면 (`result.png`)

1. 완성된 자소서 Full Review 화면
2. 문항들이 나열된 최종 결과물 스크롤 확인
3. **5초 정지**

> 📸 **이 구간에서 `result.png` 스크린샷** (2~3초 지점)

---

### [I구간] 2:20 ~ 2:40 — Chrome 확장 자동 입력 (`extension.gif`)

1. **[확장으로 전송]** 버튼 클릭 (천천히)
2. Chrome 채용 사이트 폼 화면으로 전환 — **빈 textarea들이 보이는 상태에서 2초 정지**
3. Chrome 확장 팝업 아이콘 클릭
4. **[자동 입력]** 클릭
5. **textarea들이 순차적으로 빠르게 채워지는 장면** — 끝까지 대기
6. 완료 후 **2초 정지**

> 🎬 **I구간 전체 = `extension.gif`** (2:20 ~ 2:40 추출)

```

품질: LossyGIF 85%
목표 크기: 4MB 이하
추천 툴: ScreenToGif 내장 편집기 (자르기 + GIF 저장 한번에 가능)
저장 위치: jasoseo-machine/assets/images/
```

## 편집 팁

- **로딩 구간 속도 2배**: AI 처리 대기 구간은 ScreenToGif에서 해당 프레임 선택 후 재생속도 2x 적용

- **무음 확인**: GIF는 소리 없음. 별도 설명 없어도 직관적으로 보이는 장면 선택
