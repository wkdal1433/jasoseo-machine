# 자소서 머신 — 프로젝트 최종 목표

**Version**: 1.0
**Last Updated**: 2026-03-12

---

## 핵심 미션

S급 합격 자소서 패턴을 기반으로, 사용자의 실제 경험(Episode)만을 활용해 **서류 합격률을 최대화**하는 AI 데스크탑 앱.

> **절대 원칙**: 에피소드에 없는 경험은 절대 사용하지 않는다.

---

## 전체 사용자 플로우

### 1단계: 프로필 설정 (최초 1회)

```
PDF / DOCX 파일 업로드
  → AI가 이력서를 직접 읽어 프로필 + 에피소드 자동 추출
  → 에피소드 발굴 마법사: AI 인터뷰 방식으로 누락 경험 추가
  → 결과: 완성된 사용자 프로필 + S-P-A-A-R-L 구조 에피소드 라이브러리
```

### 2단계: 지원서 작성 (매 회사마다)

```
회사명 + 직무 + 자소서 문항 입력
  → [Step 0] 기업 리서치: 채용 공고 분석 → HR 의도 2개 + 작성 전략 결정
  → [Step 1] 질문 재해석: 질문을 기업 가치에 유리하게 재정의
  → [Step 2] 에피소드 매칭: AI가 최적 에피소드 제안 → 사용자 승인
  → [Step 3-5] 실시간 AI 스트리밍 생성: S-P-A-A-R-L 두괄식 구조
  → [Step 6] 할루시네이션 검증
  → [Step 7] 탈락 패턴 제거
  → [Step 8] 이중 코딩 최종 검증 (내 이야기 + 회사 가치 언어 동시 전달)
  → 완성된 자소서 → 인라인 편집 / Surgical Edit
```

### 3단계: Chrome 확장 자동 제출

```
완성된 자소서 → Bridge Server (WebSocket)
  → Chrome 확장 → 채용 사이트 폼 자동 감지 + 입력
```

### 구현 예정 기능

#### #1. 글자 수 제한 대응 (우선순위: 높음)

Chrome 확장이 각 문항의 글자 수 제한을 추출해 전달하면,
AI가 처음부터 해당 제한 내에서 생성한다.

- Chrome 확장: 폼 분석 시 `maxlength` / 실시간 카운터 감지 → 글자 수 전달
- 위저드 Step 3-5: 프롬프트에 `charLimit` 명시 → 초과 시 자동 축약
- UI: 실시간 글자 수 카운터 + 제한 도달 시 경고 표시

#### #2. 에피소드 중복 사용 방지 (우선순위: 높음)

동일 지원서 내 여러 문항 작성 시, 이미 사용한 에피소드를 자동으로 필터링한다.

- 위저드 Step 2: 에피소드 제안 시 현재 지원서의 `usedEpisodeIds` 목록 확인
- 동일 에피소드가 3회 초과 사용될 경우 AI 프롬프트에 경고 포함
- UI: 이미 사용된 에피소드에 배지 표시

## 기술 원칙

| 원칙 | 내용 |
|------|------|
| AI Provider 분리 | Gemini CLI: 경로 중립화(한글→ASCII 임시 복사) 필요 / Claude Code: 직접 처리 가능 |
| 단일 사실 원천 | 자소서 내용 = 에피소드 파일에 명시된 내용 ONLY |
| Zero-Assumption | 추론·유추·맥락 보완 전면 금지 |
| Quote-Level Fidelity | 고유명사·수치·기간은 원문 그대로 사용 |

#### #3. 소제목 품질 등급 판정 S~C (CareerXP 벤치마크)

위저드 Step 3에서 AI가 소제목을 생성할 때 품질 등급 + 개선안을 즉시 피드백한다.

- AI가 소제목 생성 시 `grade: S|A|B|C` + `reason: string` 함께 반환
- S급 기준: 직무 키워드 + 해결 과제 + 기여 방향 3요소 모두 포함
- C급 예시: "열정 넘치는 인재" → S급: "저전력 OPAMP 설계, 전력 최적화로 개발 사이클 단축"
- UI: 등급 배지 + 한 줄 이유 + "S급으로 개선" 버튼

#### #4. 기업별 적합도 점수 산출 (CareerXP 벤치마크)

Step 0 기업 리서치 완료 후, 사용자 에피소드 라이브러리와 기업 요구 역량을 매핑해 적합도 점수를 시각화한다.

- 기업 HR 의도(Execution/Growth/Stability/Communication) 각 항목별 에피소드 보유 현황 확인
- "이 기업에 맞는 에피소드 3개 / 전체 8개" 형태의 간단한 커버리지 표시
- 부족한 HR 의도 항목은 에피소드 보강 권유 메시지 출력

#### #5. 멀티타겟 앵글 비교 뷰 (CareerXP 벤치마크)

동일 에피소드를 서로 다른 기업/직무 앵글로 작성한 결과를 좌우 비교한다.

- 에피소드 라이브러리에서 "이 에피소드로 작성된 자소서 목록" 확인
- 삼성생명용 / KB증권용 / 현대해상용 동일 에피소드 결과물 나란히 비교
- 재지원 또는 복수 지원 시 일관성 검토 용도

#### #6. 멀티 문항 일관성 검증 — 전체 자소서 크로스 검증 (필수)

동일 지원서의 여러 문항을 각각 독립 생성하면 전체를 읽었을 때 에피소드가 중복되거나 앞뒤 맥락이 충돌한다.
이를 방지하기 위해 개별 문항 완성 후 "전체 자소서 단위 일관성 검증" 단계를 추가한다.

**① 생성 시점 — 에피소드 배분 계획 수립 (Step 2 확장)**
- 위저드 시작 시 전체 문항 수를 파악하고, AI가 각 문항에 어떤 에피소드·HR 의도를 배분할지 계획표 먼저 생성
- 예: Q1(Execution, ep02) / Q2(Growth, ep05) / Q3(Stability, ep03) / Q4(Communication, ep06)
- 이후 각 문항 생성 시 이 계획표를 컨텍스트로 주입 → "Q1에서 ep02를 이미 실행력 각도로 썼으니 Q3에서는 안정성 각도로 다르게 쓸 것"

**② 검증 시점 — 전체 크로스 검증 (Step 8 이후 신규 Step 9)**
- 모든 문항 완성 후 AI가 전체 자소서를 한 번에 읽고 다음을 점검:
  - 동일 에피소드 중복 사용 여부
  - HR 의도 4가지(Execution/Growth/Stability/Communication) 균형 커버 여부
  - 수치·고유명사 불일치 (예: Q1에서 "3개월"이라 했는데 Q3에서 "6개월")
  - 전체 서사 흐름 — HR이 4문항을 연속으로 읽었을 때 동일 인물의 일관된 이야기인가
- 결과: 항목별 패스/경고 + 수정 권고 문장

#### #7. 에피소드 자동 승인 모드 (설정 옵션)

- 기본값: 수동 승인 (사용자가 에피소드 선택 확인 후 진행)
- 설정 → "AI 추천 자동 승인" 토글 ON 시: Step 2 에피소드 매칭을 건너뛰고 AI 최고 점수 에피소드 자동 선택
- 숙련 사용자 또는 빠른 초안 작성 시 활용
