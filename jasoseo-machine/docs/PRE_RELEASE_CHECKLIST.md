## 🔴 P0 — 앱이 아예 안 열리는 상황 (크리티컬)

### [ ] 1. CLI 미설치 상태에서 앱 실행
- 테스트: `claude` 또는 `gemini` 명령이 없는 환경에서 앱 실행
- 기대: 설정 페이지로 안내 또는 친절한 에러 메시지
- 실패 시: 앱이 조용히 멈추거나 의미 없는 에러 → 수정 필요

### [ ] 2. 프로젝트 디렉토리 미설정 상태 첫 실행
- 테스트: 설정에서 프로젝트 디렉토리 지정 전 모든 기능 클릭
- 기대: "설정 먼저 해주세요" 안내
- 실패 시: 앱 크래시

### [ ] 3. 한글/공백이 포함된 경로에서 실행
- 테스트: `C:\Users\사용자 이름\바탕 화면\자소서 머신\` 같은 경로에 설치
- 기대: 정상 동작
- 실패 시: CLI subprocess 경로 오류 (특히 gemini/claude 실행 시)

### [ ] 4. 앱 재시작 후 데이터 유지
- 테스트: 자소서 작성 중 앱 강제 종료 → 재실행
- 기대: 임시저장 데이터 복구 가능
- 실패 시: 데이터 손실

## 🟠 P1 — 핵심 기능 오류 (주요)

### [ ] 5. 매직 온보딩 — PDF 업로드
- 테스트: 이력서 PDF 업로드 → AI 파싱
- 기대: 프로필 + 에피소드 자동 추출
- 주의: 파일 경로에 한글/공백 포함 시 Gemini CLI 실패 가능
- 실패 시: "파일을 읽을 수 없습니다" 에러

### [ ] 6. 매직 온보딩 — DOCX 업로드
- 테스트: .docx 파일 업로드
- 기대: 텍스트 추출 후 AI 파싱
- 주의: mammoth가 구버전 .doc는 실패함 → PDF 우선 안내 필요

### [ ] 7. 자소서 생성 전체 플로우 (Step 0~8)
- 테스트: 회사명/직무/문항 입력 → AI 분석 시작 → 최종 생성까지
- 기대: 각 Step이 순서대로 진행, 스트리밍 출력 표시
- 주의: Claude CLI rate limit 시 타임아웃 발생 가능

### [ ] 8. 에피소드 없을 때 Step 2 진행
- 테스트: 에피소드 0개 상태에서 지원서 작성
- 기대: "에피소드가 없습니다" 안내 또는 건너뛰기 처리
- 실패 시: undefined 참조 오류

### [ ] 9. 자소서 저장 → 이력 페이지 확인
- 테스트: FullReview에서 "이력 저장" → 이력 페이지에서 확인
- 기대: 저장된 자소서 표시

### [ ] 10. 합격 버튼 → 패턴 추가 모달
- 테스트: 이력 페이지 → 완료 상태 항목 → 합격 클릭
- 기대: 축하 모달 표시 → 패턴 추가 동작
- 주의: 저장된 자소서 텍스트가 없으면 "내용 없음" 처리

### [ ] 11. 패턴 강화 페이지 — 파일 업로드
- 테스트: .txt/.md/.pdf 파일 업로드 → 분석 대기
- 기대: 분석 중 → 분석 완료 상태 전환
- 실패 시: 분석 실패 상태로 고정

## 🟡 P2 — UX 오류 (중요)

### [ ] 12. 다음 문항 이동 버튼
- 테스트: FinalResult 화면에서 "다음 문항 →" 클릭
- 기대: 다음 문항으로 이동, 마지막 문항에서는 버튼 미표시

### [ ] 13. 저장 완료 후 대시보드 이동
- 테스트: FullReview에서 저장 → "저장 완료 — 대시보드로 →" 클릭
- 기대: 대시보드로 이동

### [ ] 14. 임시저장 이어서 작성
- 테스트: 작성 중 다른 페이지 이동 → 이력 페이지 → 임시저장 클릭
- 기대: 작성 중이던 상태 그대로 복원

### [ ] 15. 다크모드 전환
- 테스트: 설정에서 다크/라이트 전환
- 기대: 모든 페이지 정상 표시

### [ ] 16. 패턴 강화 — 기본 패턴 토글
- 테스트: 사용자 패턴 1개 이상 활성화 → 기본 패턴 끄기 시도
- 기대: 사용자 패턴 없을 때 기본 패턴 끄기 비활성화

## 🔵 P3 — Chrome 확장 연동 (선택)

### [ ] 17. Bridge Server 시작 확인
- 테스트: 앱 실행 → 설정 페이지에서 Bridge 정보 확인
- 기대: 포트 번호 표시, ping 응답

### [ ] 18. 확장 프로그램 문항 추출
- 테스트: 사람인/잡코리아 지원서 페이지 → 확장 "문항 추출" 클릭
- 기대: 앱에 문항 자동 입력

### [ ] 19. 확장 프로그램 자동 입력
- 테스트: 완성된 자소서 → Bridge → 확장 "입력" 클릭
- 기대: 채용 사이트 필드에 텍스트 자동 입력
