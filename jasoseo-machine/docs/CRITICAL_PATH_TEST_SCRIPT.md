# Critical Path Test Script — 자소서 머신

> Phase 완료 후, 그리고 버그 수정 후 반드시 실행
> ✅ 통과 / ❌ 실패 / ⚠️ 부분 동작 으로 기록

---

## 사전 조건
- 앱 정상 실행 (`npm run dev`)
- Gemini CLI 또는 Claude CLI 연결 확인 (Settings → CLI 테스트)
- 테스트용 에피소드 최소 2개 존재

---

## 회귀 테스트 기준

```
Phase 완료 시 반드시 실행할 시나리오:
  Phase 1 완료: A, D (데이터 구조 변경 영향 확인)
  Phase 2 완료: A, B, C (UI 교체 영향 확인)
  Phase 3 완료: A, B, C, D, E 전체

버그 수정 후: 해당 시나리오 + B, C (스트림/취소 영역 회귀)
```
