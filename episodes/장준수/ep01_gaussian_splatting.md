---

## 프로젝트 핵심 가치
> **AI 모델을 '웹 서비스 형태'로 실제 사용 가능하게 연결한 End-to-End 파이프라인 구축**

이 프로젝트는 단순히 "Gaussian Splatting을 해봤다"가 아니다.
**연구용 오픈소스를 사용자 서비스 구조로 전환**한 경험이다.

---

## 시스템 아키텍처
```
Web UI ↔ Backend ↔ Local GPU AI Model ↔ 3D PLY Viewer
```

### 전체 흐름
1. **3D Gaussian Splatting 오픈소스** → 로컬(학교 컴퓨터)에 직접 구축
2. **학습 및 추론 환경 세팅** → GPU 환경 최적화
3. **웹페이지 구현** → 사용자가 영상 or 다중 시점 이미지 업로드
4. **서버 연동** → 업로드 트리거 → 학교 컴퓨터에서 AI 모델 실행
5. **결과 시각화** → PLY 기반 3D 뷰어로 웹에서 렌더링

---

## S-P-A-A-R-L 구조

### S (상황)
- 대부분의 학우들은 생성형 AI API만 사용하여 챗봇 수준의 과제를 제출
- 최신 3D AI 기술(Gaussian Splatting)은 논문/연구 수준에 머물러 있음

### P (문제 정의)
- 최신 AI 기술이 연구 단계에 머물러 일반 사용자가 체험할 수 없는 상태
- 차별화된 결과물을 만들려면 **단순 데모가 아닌 실제 서비스 형태**가 필요
- GPU 메모리와 연산 한계로 모델링 시간이 과도하게 오래 걸리는 문제

### A (분석 / 의사결정)
1. **경량화 모델 선택**: 다양한 경량화 모델(Mini Splatting 등)을 비교·테스트 → Mini Splatting 채택 (연산 시간 대비 품질 최적)
2. **배포 방식 선택**: Cloudflare CDN 활용 시도 → 대용량 PLY 파일의 캐싱 정책 충돌, 수동 빌드/업로드 이슈, 모니터링 체계 부재로 실패 → 같은 네트워크 내 자체 서버로 전환 판단
3. **추가 최적화**: 입력 영상 해상도 제한으로 품질-속도 트레이드오프 조절

### A (행동)
1. **환경 구축**: Gaussian Splatting 오픈소스를 학교 로컬 GPU 서버에 직접 설치 (학습/추론 환경 세팅)
2. **경량화 적용**: Mini Splatting 도입으로 모델링 시간 20분 → 1~2분으로 단축
3. **웹 UI 구현**: 사용자가 영상 또는 다중 시점 이미지를 업로드할 수 있는 인터페이스 개발
4. **서버 연동**: 업로드 트리거에 따른 AI 모델 자동 실행 로직 구현 (Web ↔ Local GPU Server)
5. **시각화**: PLY 기반 3D 뷰어를 웹에 연동, AI 추론 결과를 브라우저에서 인터랙티브하게 확인 가능하도록 구현
6. **파이프라인 완성**: COLMAP 전처리 → Gaussian Splatting → PLY 웹뷰어 End-to-End 파이프라인 구축

### R (결과)
- 학우들이 접속 가능한 **'3D 체험 서비스'** 런칭
- 모델링 시간 **20분 → 1~2분** 단축
- A+ 성적 획득
- **"연구 결과를 실제 서비스로 전환할 수 있는 실행력"** 증명

### L (배움 / 후속)
- AI 파이프라인은 연산, 저장, 배포, 모니터링이 유기적으로 연결된 구조라는 점을 깨달음
- Cloudflare 배포 실패를 통해 클라우드 인프라 이해의 필요성을 절실히 체감
- 이후 AI & Applications Lab(명지대 랩실)에서 6개월간 시간축 확장 연구를 진행
- 연구의 깊이와 실무 적용의 속도감·효율성 사이에서 고민 → "AI 기술의 진정한 가치는 연구 자체가 아니라 실제 문제 해결과 생산성 혁신에 있다"는 깨달음

## 기술적 구현 상세

| 영역 | 구현 내용 |
|------|----------|
| **AI/ML** | Gaussian Splatting 오픈소스 환경 구축, Mini Splatting 기법 적용 (GPU 최적화), 모델링 시간 20분→1~2분 단축 |
| **Backend** | Java 기반, 파일 업로드 처리, AI 모델 실행 트리거, 결과 파일 관리 |
| **Frontend** | 파일 업로드 UI, 진행 상태 표시, 3D 뷰어 연동 |
| **Infra** | Web ↔ Local GPU Server 연동, 비동기 처리, COLMAP→Splatting→PLY 파이프라인 |

## 자소서 작성 시 주의사항

### DO (이렇게 써라)
- "연구용 오픈소스를 **사용자 서비스 구조로 전환**한 경험"
- "Web ↔ AI Model ↔ 3D Viewer **End-to-End 파이프라인 설계**"
- "모델링 시간 **20분→1~2분** 단축" (구체적 수치 반드시 포함)
- "Cloudflare 배포 실패 후 **자체 서버 전환 판단**" (의사결정 과정 강조)
- "모델 성능보다 **서비스화 과정의 판단**을 강조"

### DON'T (이렇게 쓰지 마라)
- "3D AI 모델을 구현했다" (너무 추상적)
- "Gaussian Splatting을 공부했다" (학습이 아닌 서비스화가 핵심)
- 기술 스택 나열만 하기
- "빠르게 해결했다" (의사결정 과정 생략 감점)
