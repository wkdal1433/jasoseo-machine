|------|------|

## 활동 핵심 가치
> **AI 모델을 블랙박스가 아닌 구조적으로 이해하는 기반 확보**
> 전역 직후 스스로 AI 분야에 뛰어들어, 이론과 실습을 병행한 자기주도 학습의 출발점

---

## S-P-A-A-R-L 구조

### S (상황)
- 2022년 12월 전역 직후, AI 기술의 급격한 발전을 인식
- AI 모델을 단순히 '사용'하는 수준이 아니라, 구조와 원리를 이해하고자 함
- 단순 API 호출이 아닌 **모델 내부 동작** 이해의 필요성을 인식하고 연세대 AI/DATA 동아리에 참여

### P (문제 정의)
- 대부분의 학습자가 AI 모델을 블랙박스로 사용하는 수준에 머물러 있음
- 논문 기반으로 모델 구조를 분석하고 직접 구현할 수 있는 역량이 필요

### A (분석)
- 이론만으로는 부족하고, 논문 분석 → 직접 구현 → 실험의 End-to-End 학습이 필요함을 판단
- 경진대회 참가를 통해 실전 감각까지 확보하는 것이 효과적임을 인식

### A (행동)
- **머신러닝**: XGBoost, LightGBM 실습, Gradient Boosting 원리 이해
- **딥러닝**: CNN, ResNet, VGG 구조 이론 및 직접 구현
- **생성모델**: Diffusion Model 이론 학습, U-Net 구조 분석 및 실험
- **Dacon 경진대회 참가**: 실전 데이터셋으로 모델 학습·평가 경험
- 스터디 내 발표 및 토론 참여

### R (결과)
- 모델 구조에 대한 **해석력 향상** — AI 알고리즘에 대한 기술적 대화 가능 수준 확보
- **GIGO(Garbage In, Garbage Out) 원칙** 습득 → Episode 03(코드비전 데이터 품질 관리)의 이론적 기반
- 논문 기반 모델 분석 → 직접 구현까지 **End-to-End 학습 경험** 축적
- Dacon 경진대회를 통한 실전 경험

### L (배움)
- AI 모델의 성능은 알고리즘 자체보다 **데이터 품질에 의존**한다는 GIGO 원칙을 이론적으로 학습 — 이후 코드비전(ep03)에서 9개월간 실무로 내재화
- 전역 직후 자기주도적으로 AI 분야에 진입한 경험이 이후 모든 프로젝트(Gaussian Splatting, DayScript 등)의 기술적 기반이 됨

## 자소서 작성 시 주의사항

### DO (이렇게 써라)
- "**전역 직후** AI 분야에 자기주도적으로 진입"
- "모델 구조를 **논문 기반으로 분석하고 직접 구현**"
- "**Dacon 경진대회** 참가로 실전 경험까지 연결"
- "GIGO 원칙을 이론으로 학습 → **코드비전 실무에서 9개월간 내재화**(ep03 연결)"
- "R&D / 기술 기반 직무에 **빠르게 적응 가능**한 학습형 인재"

### DON'T (이렇게 쓰지 마라)
- "동아리에서 AI 공부했다" (무엇을, 어떻게 공부했는지가 중요)
- "CNN, ResNet 알아요" (아는 것과 구조를 이해하고 구현하는 것은 다름)
- 전역 직후라는 맥락 없이 서술 (자기주도성이 핵심 포인트)
