## 프로젝트 핵심 가치
> **실무 환경에서 결과물을 만들어내는 상용 서비스 배포 경험**

---

## STAR 구조

### S (상황)
- YTN에서 연말 특집 콘텐츠를 위한 인터랙티브 웹 페이지 외주 의뢰
- 콘텐츠 몰입도를 극대화하는 UI 요구

### T (과제)
- 스크롤에 따라 카드가 동적으로 반응하는 UI 구현
- 퍼포먼스 최적화 (로딩 속도, 렌더링 효율)

### A (행동)
- **스크롤 트리거 기반 카드형 레이아웃** 설계
- **사용자 흐름(Storytelling) 중심** 구성
- **퍼포먼스 최적화**: 불필요한 리렌더링 방지
- 연말 특집 성격에 맞춘 **시각적 연출**

### R (결과)
- 상용 서비스로 실제 배포 완료
- YTN 대상 **B2B 실무 외주 경험** 획득

---

## HR 관점 해석

| HR 의도 | 이 경험이 증명하는 것 |
|---------|---------------------|
| **Execution** | 실무 환경에서 결과물을 만들어내는 인재 |
| **Stability** | B2B 클라이언트와의 협업 신뢰성 |

---

## 자소서 작성 시 주의사항

### DO (이렇게 써라)
- "외주가 아닌 **상용 서비스 배포 경험**으로 포지셔닝"
- "스크롤 기반 **인터랙티브 UI 구현**"
- "**퍼포먼스 최적화** (리렌더링 방지 등) 기술적 깊이"

### DON'T (이렇게 쓰지 마라)
- "외주 했다" (상용 배포가 핵심)
- "웹페이지 만들었다" (인터랙티브/퍼포먼스가 차별점)
