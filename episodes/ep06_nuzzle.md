## 서비스 소개
> **가족 특화 소규모 커뮤니티 앱**

- **컨셉**: 가족 같은 소규모 단위의 그룹을 만들어 소규모 커뮤니티 활성화
- **벤치마킹**: 인스타그램과 유사한 UX (피드, 게시물, 좋아요 등)
- **차별점**: 가족/친밀한 관계 중심의 프라이빗 커뮤니티

---

## 프로젝트 핵심 가치
> **타이트한 일정 속에서 품질과 일정을 동시에 맞추는 실행력**

## S-P-A-A-R-L 구조

### S (상황)
- UMC 6기 프로젝트로 **2개월 내 서비스 런칭** 목표
- 프론트엔드, 앱, 백엔드 팀이 동시에 개발 진행
- UMC 파트장으로서 '너즐' 프로젝트의 Spring 백엔드 개발자로 참여

### P (문제 정의)
- 2개월이라는 짧은 기간 안에 REST API 설계부터 구현·연동까지 완료해야 하는 일정 압박
- 프론트엔드/앱 팀과 API 스펙이 불일치하면 전체 일정이 지연되는 구조

### A (분석)
- 주간 단위 마일스톤을 설정하여 진행 상황을 가시화하고 지연을 조기 감지하는 체계 필요
- 프론트/앱 팀과의 API 스펙 조율을 선행해야 구현 단계에서의 재작업을 방지할 수 있음을 판단

### A (행동)
- **백엔드 설계 및 구현**: 회원 인증, 가족 그룹 관리, 초대 시스템, 사진/이모지 API 구현
- **테이블 설계**: 회원, 가족 그룹, 사진, 이모지 테이블 설계 및 JPA 기반 CRUD 구현
- **API 스펙 조율**: 프론트엔드/앱 팀 요구사항을 반영하여 REST API 설계, 지속적 커뮤니케이션으로 스펙 불일치 방지
- **일정 관리**: 주간 단위 마일스톤 설정 및 준수
- **Android 동시 경험**: Kotlin 기반 Android 파트도 경험 (MVVM 패턴, Retrofit 기반 통신)

### R (결과)
- 2개월 내 프로젝트 완료 및 서비스 배포 성공
- 크로스팀(프론트/앱/백엔드) 협업 경험 축적
- 백엔드와 Android 양쪽을 경험하며 풀스택 관점 확장

### L (배움)
- 타이트한 일정에서는 API 스펙 조율을 선행하는 것이 전체 효율에 결정적임을 체감
- 백엔드와 Android를 동시에 경험하며 클라이언트-서버 간 데이터 흐름을 통합적으로 이해
