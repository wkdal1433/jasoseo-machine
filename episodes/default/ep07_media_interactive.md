## 프로젝트 핵심 가치
> **실무 환경에서 상용 서비스를 단독으로 배포한 B2B 외주 경험**

---

## S-P-A-A-R-L 구조

### S (상황)
- YTN에서 연말 특집 '올해의 뉴스' 콘텐츠를 위한 인터랙티브 웹 페이지 외주 의뢰
- 콘텐츠 몰입도를 극대화하는 동적 UI가 핵심 요구사항

### P (문제 정의)
- 정적 페이지로는 연말 특집 콘텐츠의 몰입감을 전달하기 어려움
- 스크롤에 따라 동적으로 반응하면서도 퍼포먼스가 저하되지 않는 UI 필요
- 단독 작업으로 기획 의도 해석부터 구현·배포까지 전 과정을 책임져야 하는 상황

### A (분석)
- 스크롤 트리거 방식이 콘텐츠 스토리텔링에 가장 적합한 인터랙션 패턴임을 판단
- 퍼포먼스 최적화를 위해 불필요한 리렌더링을 방지하는 설계가 필수

### A (행동)
- **스크롤 트리거 기반 카드형 레이아웃** 설계 및 구현
- **사용자 흐름(Storytelling) 중심** UI 구성 — 스크롤에 따라 카드가 동적으로 반응
- **퍼포먼스 최적화**: 불필요한 리렌더링 방지
- 연말 특집 성격에 맞춘 **시각적 연출** 구현

### R (결과)
- **상용 서비스로 실제 배포 완료** — YTN 사이트에서 서비스
- YTN 대상 **B2B 실무 외주 경험** 획득
- 기획 의도 해석 → 디자인 구현 → 배포까지 **단독으로 전 과정 완수**

### L (배움)
- B2B 클라이언트의 요구사항을 정확히 해석하고 기한 내 결과물을 전달하는 실무 감각 습득
- 상용 서비스 배포의 전 과정을 단독으로 경험하며 결과물에 대한 책임감 체득

## HR 관점 해석

| HR 의도 | 이 경험이 증명하는 것 |
|---------|---------------------|
| **Execution** | 기획 해석 → 구현 → 상용 배포까지 단독 완수하는 실행력 |
| **Stability** | B2B 클라이언트(YTN)와의 협업 신뢰성, 기한 내 납품 |

## 자소서 작성 시 주의사항

### DO (이렇게 써라)
- "YTN 대상 **상용 서비스 배포** 경험으로 포지셔닝"
- "스크롤 기반 **인터랙티브 UI** — 스토리텔링 중심 설계"
- "**퍼포먼스 최적화** (리렌더링 방지) 기술적 깊이"
- "기획 해석 → 구현 → 배포까지 **100% 단독 완수**"
- "**B2B 실무 외주** 경험 (대형 언론사 클라이언트)"

### DON'T (이렇게 쓰지 마라)
- "외주 했다" (상용 배포가 핵심)
- "웹페이지 만들었다" (인터랙티브/퍼포먼스가 차별점)
- 클라이언트명(YTN) 없이 "외주" (구체성 부족)
