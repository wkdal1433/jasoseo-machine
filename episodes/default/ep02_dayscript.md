## S-P-A-A-R-L 구조

### S (상황)
- AI Agent(Claude Code)와 바이브 코딩(Vibe Coding)이라는 개발 패러다임 변화를 빠르게 포착
- "기술을 아는 기획자 + AI = 무한 생산성" 가설 수립

### P (문제 정의)
- "개발 지식을 갖춘 기획자가 AI와 협업할 때 생산성의 한계는 없다"는 가설을 검증할 실증 사례가 필요
- 기존 개발 프로세스(기획→디자인→개발)는 수 주가 소요되는 비효율 구조

### A (분석 / 의사결정)
1. **AI 도구 선택**: Claude Code 채택 (이유: MCP 연동으로 기획→디자인→구현 파이프라인 자동화 가능)
2. **프레임워크**: React Native 선택 (이유: 크로스 플랫폼 + AI 코딩 친화적)
3. **워크플로우 설계**: 직접 코딩 vs PM형 감독 → PM형 선택 (AI에 구현 위임, 오류/방향만 관리)

### A (행동)
- Claude Code로 목업 디자인 생성 → Figma에서 색상/배치 조정하여 디자인 완성
- Figma MCP로 디자인 링크를 연동해 Claude Code로 구현
- GitHub MCP로 기능별 커밋을 자동 관리
- PM처럼 오류 처리와 수정 사항만 결정하는 워크플로우 구축
- React Native 기반 안드로이드 구현 (처음임에도 기존 방식보다 효율적)
- 레벨/포인트 시스템, 학습 기록 관리 구현

### R (결과)
- 기존 **수 주가 걸리던 개발 과정을 1주일로** 단축
- AI 에이전트를 활용한 바이브 코딩 워크플로우로 **개발 효율 70% 이상** 향상
- 1인 풀스택 개발 생산성 실증

### L (배움 / 한계)
- 기획→설계→구현 전 과정을 AI와 협업해 완주할 수 있음을 검증
- 디자인과 백엔드/프론트 코드까지는 검증하며 구축할 수 있었으나, **클라우드 MCP 워크플로우는 클라우드 관련 지식 부족으로 미완성** (한계 인식)
- 이 한계가 클라우드 학습의 동기가 됨

## 자소서 작성 시 주의사항

### DO (이렇게 써라)
- "기존 **수 주가 걸리던 과정을 1주일로** 단축" (정량적 근거 필수)
- "기획자가 개발까지 할 수 있는 시대에 대한 **통찰 + 실증**"
- "Figma/GitHub/Notion **MCP 3종 연동**으로 파이프라인 자동화"
- "PM처럼 **오류와 방향만 관리**하는 워크플로우" (역할 전환 강조)
- 한계 인식도 함께 서술 (클라우드 MCP 미완성 → 성장 동기)

### DON'T (이렇게 쓰지 마라)
- "AI 코딩 도구를 써봤다" (경험이 아닌 실험/검증이 핵심)
- "1인 개발 했다" (왜 1인으로 했는지, 무엇을 검증했는지가 중요)
- "효율이 좋았다" (70% 이상이라는 수치 없이 추상적 표현 금지)
