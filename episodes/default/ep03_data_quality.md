# Episode 03. 데이터 품질 관리 (코드비전)

## 기본 정보
| 항목 | 내용 |
|------|------|
| **프로젝트명** | AI 학습 데이터 라벨링 및 검수 |
| **소속** | (주)코드비전 |
| **기간** | 2024.10 ~ 2025.06 (약 9개월) |
| **역할** | 데이터 라벨러 / 품질 검수 (QA) |
| **고용형태** | 계약직 |
| **관련 학습** | 연세대 AI/DATA 동아리에서 GIGO 원칙 습득 |

---

## 프로젝트 핵심 가치
> **AI 모델 성능은 데이터 품질에 의존한다 (GIGO: Garbage In, Garbage Out)**
> 이 원칙을 이론이 아닌 **실무에서 체화**한 경험

---

## S-P-A-A-R-L 구조

### S (상황)
- 연세대 동아리에서 'GIGO(Garbage In, Garbage Out)' 원리를 배움
- AI 모델 성능이 데이터 품질에 크게 의존함을 이론적으로 이해

### P (문제 정의)
- AI 학습용 데이터의 작은 오류가 모델 성능을 직접적으로 저하시킴
- 고객 데이터 하나하나가 서비스 품질을 결정한다는 현실

### A (분석)
- 동아리에서 배운 GIGO 이론을 실무 QA 마인드로 전환할 필요성 인식
- 데이터 무결성이 모든 IT 시스템의 근본임을 판단

### A (행동)
- **1px 단위 오차**까지 허용하지 않는 집요한 검수
- **태깅 불일치, 라벨 노이즈** 철저히 제거
- 학습 데이터의 오류·노이즈를 최소화하여 모델 신뢰도 확보

### R (결과)
- 데이터 무결성 확보
- "데이터를 다루는 기획자/개발자의 기본 태도" 완성
- **리스크 관리형 인재** 이미지 구축

### L (배움)
- "Garbage In, Garbage Out"이라는 데이터의 본질을 실무에서 체득
- 이론으로 알던 것을 9개월간 반복 실무로 내재화
- 이 가치를 실무 품질 검수(QA) 역량으로 발전시킴

## HR 관점 해석

| HR 의도 | 이 경험이 증명하는 것 |
|---------|---------------------|
| **Stability** | 보이지 않는 품질까지 책임지는 신뢰 기반 인재 |
| **Execution** | 운영 환경에서 실수를 줄이는 리스크 관리 능력 |

## 자소서 작성 시 주의사항

### DO (이렇게 써라)
- "AI 모델 성능이 데이터 품질에 의존한다는 것을 **9개월 실무에서 체감**"
- "1px 단위 오차도 허용하지 않는 **집요한 품질 관리**"
- "GIGO 원칙을 이론(동아리) → **실무 태도(코드비전)**로 내재화한 과정"
- 금융/보험 기업에는 "고객 데이터 하나하나가 서비스 품질을 결정" 연결

### DON'T (이렇게 쓰지 마라)
- "라벨링 했다" (단순 업무 나열)
- "데이터 검수 경험이 있다" (왜 중요한지, 어떻게 했는지가 중요)
