## 프로젝트 핵심 가치
> **AI 모델을 '웹 서비스 형태'로 실제 사용 가능하게 연결한 End-to-End 파이프라인 구축**

이 프로젝트는 단순히 "Gaussian Splatting을 해봤다"가 아니다.
**연구용 오픈소스를 사용자 서비스 구조로 전환**한 경험이다.

---

## 시스템 아키텍처
```
Web UI ↔ Backend ↔ Local GPU AI Model ↔ 3D PLY Viewer
```

### 전체 흐름
1. **3D Gaussian Splatting 오픈소스** → 로컬(학교 컴퓨터)에 직접 구축
2. **학습 및 추론 환경 세팅** → GPU 환경 최적화
3. **웹페이지 구현** → 사용자가 영상 or 다중 시점 이미지 업로드
4. **서버 연동** → 업로드 트리거 → 학교 컴퓨터에서 AI 모델 실행
5. **결과 시각화** → PLY 기반 3D 뷰어로 웹에서 렌더링

---

## STAR 구조

### S (상황)
- 대부분의 학우들은 생성형 AI API만 사용하여 챗봇 수준의 과제를 제출
- 최신 3D AI 기술(Gaussian Splatting)은 논문/연구 수준에 머물러 있음

### T (과제)
- 차별화된 결과물: 최신 3D 기술을 학우들이 **직접 체험**할 수 있게 하고 싶었음
- 단순 데모가 아닌 **실제 서비스 형태**로 제공하고자 함

### A (행동)
1. **환경 구축**: Gaussian Splatting 오픈소스를 학교 로컬 GPU 서버에 직접 설치 (학습/추론 환경 세팅)
2. **웹 UI 구현**: 사용자가 영상 또는 다중 시점 이미지를 업로드할 수 있는 인터페이스 개발
3. **서버 연동**: 업로드 트리거에 따른 AI 모델 자동 실행 로직 구현 (Web ↔ Local GPU Server)
4. **시각화**: PLY 기반 3D 뷰어를 웹에 연동, AI 추론 결과를 브라우저에서 인터랙티브하게 확인 가능하도록 구현
5. **UX 고려**: 비전공자도 사용 가능한 직관적 UI 설계

### R (결과)
- 학우들이 접속 가능한 **'3D 체험 서비스'** 런칭
- A+ 성적 획득
- **"연구 결과를 실제 서비스로 전환할 수 있는 실행력"** 증명
