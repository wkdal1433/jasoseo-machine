## 프로젝트 핵심 가치
> **서비스 운영을 고려한 구조적 사고**: 보안, 확장성, 유지보수성을 동시에 설계

---

## STAR 구조

### S (상황)
- 로컬 저장소의 한계 및 보안 취약점 발견
- 팀 프로젝트가 커지면서 코드 유지보수 어려움 증가

### T (과제)
- 엔터프라이즈급 보안 체계 구축
- 확장 가능한 저장소 아키텍처로 전환
- 유지보수 가능한 코드 구조 확립

### A (행동)
- **보안**: Spring Security + JWT 기반 인증/인가 시스템 구축
- **확장성**: 로컬 저장소 → AWS S3 마이그레이션
- **유지보수**: SOLID 원칙 기반 코드 리팩토링

### R (결과)
- 엔터프라이즈급 안정성 확보
- B2B 서비스 기획의 기술적 기반 마련
- 팀 내 코드 리뷰 문화 정착

## 자소서 작성 시 주의사항

### DO (이렇게 써라)
- "로컬 → 클라우드 전환의 **의사결정 과정**"
- "보안, 확장성, 유지보수성을 **동시에 고려한 설계**"
- "SOLID 원칙을 **실제 프로젝트에 적용**"

### DON'T (이렇게 쓰지 마라)
- 기술 스택만 나열 (Spring, AWS... 등)
- "백엔드 개발했다" (왜 그 선택을 했는지가 중요)
