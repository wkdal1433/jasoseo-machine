# Fail Patterns (탈락 패턴 정의)
# S급 합격 자소서에서 절대 발견되지 않는 패턴

---

## 1. 탈락 패턴 목록

| 탈락 패턴 | 설명 | 채용담당자 반응 |
|----------|------|----------------|
| **자화자찬** | "저는 뛰어난/우수한/탁월한 사람입니다" | "증거 없이 주장만" |
| **기술 스택 나열** | "Python, Java, Spring, Docker, AWS..." | "그래서 뭘 했는데?" |
| **회사 정보 복붙** | "귀사는 업계 1위로서..." | "이건 내가 더 잘 알아" |
| **막연한 포부** | "열심히 하겠습니다", "최선을 다하겠습니다" | "구체적으로 뭘 할 건데?" |
| **감정 과잉** | "정말 힘들었지만", "눈물이 났습니다" | "감정이 아니라 행동이 궁금해" |
| **남 탓** | "팀원이 안 해서", "환경이 열악해서" | "이 사람은 핑계를 대는 사람" |
| **본인 역할 불분명** | "저희 팀은 ~했습니다" | "그래서 네가 한 게 뭔데?" |
| **결과만 나열** | "성공했습니다", "수상했습니다" | "과정이 안 보여" |
| **~라고 생각합니다 남발** | 매 문장 끝에 반복 | "자신감 없어 보임" |

---

## 2. 금지 표현 상세

### 2.1 도입부 금지 패턴

| 금지 | 이유 |
|------|------|
| "저는 ~한 사람입니다"로 시작 | 자화자찬, 증거 없음 |
| 회사 소개로 시작 | 채용담당자는 이미 알고 있음 |
| 너무 긴 도입부 (5줄 이상) | 두괄식 효과 상실 |

### 2.2 마무리 금지 패턴

| 금지 | 이유 |
|------|------|
| "열심히 하겠습니다" | 구체성 없음, 누구나 쓰는 표현 |
| "최선을 다하겠습니다" | 행동이 보이지 않음 |
| "배우겠습니다"만 | 회사는 학원이 아님 |
| 회사 비전 복붙 | 진정성 없음 |

### 2.3 기술 서술 금지 패턴

| ❌ 탈락 패턴 | ✅ 합격 패턴 |
|------------|------------|
| "Docker를 사용해 컨테이너화했습니다" | "Docker Network를 도입해 포트 노출을 최소화하며 내부 통신과 보안성을 안정적으로 확보했습니다" |
| "Prometheus와 Grafana를 활용했습니다" | "문제의 근본 원인을 밝히기 위해 Prometheus와 Grafana 기반 모니터링 환경을 구축하고 데이터를 분석했습니다" |
| "RabbitMQ를 도입했습니다" | "서버 다운 시에도 데이터 유실 없이 복구되도록 RabbitMQ를 도입해 메시지를 디스크에 기록하고 DLQ로 재처리할 수 있게 개선했습니다" |

---

## 3. 감정 서사 사용 규칙

| 허용 | 금지 |
|------|------|
| 회사 가치와 연결될 때 ("안심", "신뢰", "성장") | 감정 과잉 ("정말 힘들었지만", "눈물이 났습니다") |
| 도입부와 마무리에서 간결하게 | 본문 전체를 감정으로 채우기 |
| 구체적 행동의 동기로서 | 결과 대신 감정으로 마무리 |

---

## 4. 탈락 패턴 체크리스트

작성 완료 후 아래 항목 점검:

- [ ] 탈락 패턴(섹션 1)에 해당하는 표현이 없는가?
- [ ] 두괄식 도입부가 3줄 이내인가?
- [ ] 마무리에서 회사 핵심 가치가 연결되어 있는가?
- [ ] "저희 팀은"이 아닌 "저는"으로 역할이 명확한가?
- [ ] 기술이 "도구"가 아닌 "문제 해결 수단"으로 서술되었는가?
