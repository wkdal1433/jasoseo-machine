## C.1 [KB증권] 가치 오버레이 규칙

### 암묵적 가치 코드
`신뢰-기반-안정성-설계`
> "보이지 않는 구조적 문제를 파고들어 신뢰를 설계한다"

### 핵심 키워드 매핑 표

| 인재상 키워드 | 문장 역할 | 적용 위치 |
|--------------|----------|----------|
| **전문성** | 문제의 근본 원인을 추적하는 태도 | Action 단계 |
| **고객중심** | 고객 자산/거래 안정성과 연결 | Result, 마무리 |
| **혁신주도** | 새로운 기술 도입의 '안정성 기여' 관점 | Analysis |
| **신뢰정직** | 정량적 근거 기반 판단 | Problem, Action |
| **동반성장** | 팀 협업을 통한 문제 해결 | Action, Learning |

### 기업 가치 반영 규칙

| 단계 | 반영 규칙 |
|------|----------|
| **두괄식** | 결론 문장에 "안정성", "신뢰성", "근본" 중 1개 이상 반영 |
| **Problem** | "시스템 신뢰성을 해치는" 또는 "고객에게 영향을 줄 수 있는" 관점으로 정의 |
| **Analysis** | 의사결정 기준을 "정량적 지표", "구조적 원인 파악"으로 연결 |
| **Action** | 행동의 목적이 "개인 성과"가 아닌 "시스템 안정성 확보"임을 강조 |
| **Result/Learning** | "신뢰성 향상", "안정적 운영 기반 마련" 등 가치 언어로 재해석 |
| **마무리** | "안정적이고 신뢰받는 금융 시스템"을 반드시 언급 |

### 감점 방지 체크

- [ ] "빠르게 해결했다"만 강조하고 근본 원인 추적 과정 생략 → 감점
- [ ] 기술 도입의 "혁신성"만 강조하고 "안정성 기여" 누락 → 감점
- [ ] 개인 성과 중심 서술 (팀/조직 관점 부재) → 감점
- [ ] "고객 자산", "거래 안정성" 연결 없이 마무리 → 감점

### 선호/비선호 표현 기준

| 구분 | 판단 기준 |
|------|----------|
| **선호** | 구조적 접근, 정량적 근거, 신뢰성/안정성 관점, 장기적 시스템 영향 고려 |
| **비선호** | 속도 중심, 단기 성과 강조, 개인 역량 과시, 기술 자랑 |

## C.2 [삼성생명] 가치 오버레이 규칙

### 암묵적 가치 코드
`안심-동반-성장`
> "안심이 성장을 가능하게 한다"

### 핵심 키워드 매핑 표

| 인재상 키워드 | 문장 역할 | 적용 위치 |
|--------------|----------|----------|
| **도전** | 한계를 미리 규정하지 않는 태도 | Problem, Action |
| **협업** | 팀원과 관점 공유, 함께 해결 | Action, Learning |
| **전문성** | 깊이 파고드는 집요함 | Analysis, Action |
| **고객 안심** | 안정적 서비스 → 고객 안심 연결 | Result, 마무리 |
| **성장** | 경험을 통한 역량 확장 | Learning |

### 기업 가치 반영 규칙

| 단계 | 반영 규칙 |
|------|----------|
| **두괄식** | "안심", "성장", "한계 확장" 중 1개 이상 반영 |
| **Problem** | "미리 한계를 규정"하거나 "현실에 타협"하는 상황을 문제로 설정 |
| **Analysis** | 판단 기준에 "더 안정적이고 나은 방법"을 모색하는 자세 포함 |
| **Action** | 행동의 동기가 "안주하지 않고 파고드는 태도"임을 강조 |
| **Result/Learning** | "안심 제공", "협업의 가치", "성장 가능성" 언어로 재해석 |
| **마무리** | "고객이 안심할 수 있는 서비스"를 반드시 언급 |

### 감점 방지 체크

- [ ] 혼자 해결한 것처럼 서술 (협업 관점 부재) → 감점
- [ ] "쉬운 방법"을 선택한 것처럼 보이는 서술 → 감점
- [ ] 결과만 나열하고 "안심/성장" 연결 없음 → 감점
- [ ] 기술적 해결만 강조하고 고객 관점 누락 → 감점
- [ ] "업계 1위" 등 회사 정보 복붙 → 감점

### 선호/비선호 표현 기준

| 구분 | 판단 기준 |
|------|----------|
| **선호** | 끈기/집요함, 협업 속 성장, 안정성 → 안심 연결, 한계 돌파 자세 |
| **비선호** | 개인 영웅담, 타협적 해결, 결과 중심 서술, 감정 과잉 |

## C.3 [한국자금중개] 가치 오버레이 규칙

### 암묵적 가치 코드
`구조적-신뢰-설계`
> "본질을 파악하고 구조로 신뢰를 만든다"

### 핵심 키워드 매핑 표

| 인재상 키워드 | 문장 역할 | 적용 위치 |
|--------------|----------|----------|
| **신뢰** | 시스템 신뢰성 기반 구축 | Result, 마무리 |
| **안정성** | 장애 상황에서도 유지되는 안정성 | Problem, Action |
| **구조 이해** | 흐름과 맥락으로 파악하는 능력 | Analysis |
| **글로벌 비전** | 금융시장 선도 기여 | 마무리 |
| **학습** | 사전 학습과 실습 투자 | Action, Learning |

### 기업 가치 반영 규칙

| 단계 | 반영 규칙 |
|------|----------|
| **두괄식** | "신뢰", "안정성", "구조" 중 1개 이상 반영 |
| **Problem** | 표면적 문제가 아닌 "본질적 문제"를 정의 (예: 서버 다운 ≠ 문제, 데이터 비영속성 = 문제) |
| **Analysis** | "흐름과 맥락 이해", "구조적 접근"을 판단 기준으로 설정 |
| **Action** | 행동의 기준이 "단편적 해결"이 아닌 "구조적 개선"임을 강조 |
| **Result/Learning** | "안정성과 가용성 향상", "신뢰 기반 시스템" 언어로 재해석 |
| **마무리** | "금융중개시장 선도", "글로벌 비전 기여"를 반드시 언급 |

### 감점 방지 체크

- [ ] 표면적 증상만 해결하고 본질 파악 과정 생략 → 감점
- [ ] 단편적 기술 적용만 서술 (구조적 이해 부재) → 감점
- [ ] "신뢰", "안정성" 키워드 없이 마무리 → 감점
- [ ] 학습/성장 과정 없이 결과만 나열 → 감점

### 선호/비선호 표현 기준

| 구분 | 판단 기준 |
|------|----------|
| **선호** | 본질 파악, 구조적 접근, 사전 학습, 안정성/가용성 관점, 중립적 신뢰 |
| **비선호** | 임시방편, 표면적 해결, 속도 우선, 기술 나열 |

## C.4 [현대해상] 가치 오버레이 규칙

### 암묵적 가치 코드
`공동체적-안심-실현`
> "공동체의 문제를 내 문제로 인식하고 안심을 만든다"

### 핵심 키워드 매핑 표

| 인재상 키워드 | 문장 역할 | 적용 위치 |
|--------------|----------|----------|
| **안심** | 고객이 체감하는 안심 | Result, 마무리 |
| **협업/존중** | 다양성 존중, 합의 도출 | Action |
| **도전** | 전례 없는 일에 과감히 도전 | Problem, Action |
| **책임** | 공동체 문제를 내 문제로 인식 | Problem |
| **혁신** | 안정성 위에 혁신 추구 | Learning, 마무리 |

### 기업 가치 반영 규칙

| 단계 | 반영 규칙 |
|------|----------|
| **두괄식** | "안심", "공동체", "도전" 중 1개 이상 반영 |
| **Problem** | "남의 일이 아닌 우리 모두의 문제"로 인식하는 관점 포함 |
| **Analysis** | 판단 기준에 "존중", "객관적 근거", "공동체 영향" 포함 |
| **Action** | 행동의 동기가 "개인 성과"가 아닌 "공동체 안심"임을 강조 |
| **Result/Learning** | "안심 제공", "안정성 + 혁신" 언어로 재해석 |
| **마무리** | "고객이 체감하는 안심", "안정성과 혁신을 동시에"를 반드시 언급 |

### 감점 방지 체크

- [ ] 개인 성과만 강조 (공동체 관점 부재) → 감점
- [ ] 갈등/의견 차이를 "틀림"으로 표현 → 감점
- [ ] 안정성 없이 혁신만 강조 → 감점
- [ ] "안심" 키워드 없이 마무리 → 감점
- [ ] 도전을 "무모함"으로 표현 → 감점

### 선호/비선호 표현 기준

| 구분 | 판단 기준 |
|------|----------|
| **선호** | 공동체 인식, 존중 기반 협업, 안심 제공, 안정성 + 혁신 균형 |
| **비선호** | 개인 영웅담, 갈등 회피, 혁신만 강조, 도전을 무모함으로 표현 |

## C.5 가치 오버레이 적용 프로토콜

### 적용 순서

> **마스터 워크플로우는 `CLAUDE.md`의 9-Step Mandatory Workflow를 따른다.**
> 이 문서(APPENDIX_C)는 다음 시점에 적용된다:

| 적용 시점 | 적용 내용 |
|----------|----------|
| Step 0 (기업 전략 해석) | 기업의 암묵적 가치 코드 확인 |
| Step 2 (앵글 결정) | 가치 코드와 Episode의 매칭 검증 |
| Step 3~5 (작성) | 기업별 톤/키워드 오버레이 적용 + 감점 방지 체크 |

### 범용 패턴과 오버레이의 역할 분담

| 문서 | 역할 | 예시 |
|------|------|------|
| `generic_storytelling_patterns.md` | **구조** (What) | "문제를 시스템 관점으로 정의" |
| `APPENDIX_C (이 문서)` | **톤/키워드** (How) | KB증권: "신뢰성", "고객 자산" |

### Episode 매칭 검증

| 매칭 결과 | 판단 기준 | 조치 |
|----------|----------|------|
| **완전 일치** | Episode의 의사결정 기준이 기업 가치 코드와 동일 | 그대로 사용 |
| **부분 일치** | 일부 기준만 일치, 나머지는 중립 | 일치하는 부분 강조, 나머지는 톤 조정 |
| **충돌 가능성** | Episode의 가치 기준이 기업 가치와 상충 | Episode 변경 또는 관점 재해석 필요 |

## C.6 신규 기업 가치 오버레이 생성 규칙

새로운 기업의 인재상/핵심 가치 제공 시:

1. **가치 추론 레이어 실행** - 인재상 키워드에서 암묵적 가치 코드 도출
2. **핵심 키워드 매핑 표 생성** - 각 키워드의 문장 역할 정의
3. **기업 가치 반영 규칙 작성** - 단계별 적용 규칙
4. **감점 방지 체크 작성** - 3~5개 항목
5. **선호/비선호 표현 기준 정의**
