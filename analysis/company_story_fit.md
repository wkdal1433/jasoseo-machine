## 2. 기업별 가치 유형 및 핵심 서사 패턴

## 3. 범용 스토리텔링 공식

### 3.1 가치 유형별 서사 변환 규칙

| 가치 유형 | 문제 설정 규칙 | 행동 설명 규칙 | 결과 해석 규칙 |
|----------|---------------|---------------|---------------|
| **안정성 중심** | 문제를 "시스템 리스크"로 정의 | 행동 목적을 "반복 방지/사전 대비"로 설명 | 결과를 "안정성 확보"로 귀결 |
| **신뢰 중심** | 문제를 "신뢰성 저해 요인"으로 정의 | 판단 기준을 "정량적/객관적"으로 명시 | 결과를 "신뢰 기반 마련"으로 귀결 |
| **성장 중심** | 문제를 "현재 방식의 안주/타협"으로 정의 | 행동 목적을 "더 나은 방법 탐색"으로 설명 | 결과를 "성장 가능성 확보"로 귀결 |
| **협업 중심** | 문제를 "팀/조직 차원의 과제"로 정의 | 행동 주체를 "나"에서 "우리"로 전환 | 결과를 "함께 만든 가치"로 귀속 |
| **본질 파악 중심** | 문제를 "표면 증상 ≠ 구조적 본질"로 구분 | 행동 목적을 "구조적 해결"로 설명 | 결과를 "본질적 개선"으로 귀결 |
| **공동체 중심** | 문제를 "나의 문제"가 아닌 "우리 모두의 문제"로 정의 | 행동 동기를 "공동체를 위해 나섰다"로 설명 | 결과를 "공동체에 미친 긍정적 영향"으로 귀결 |
| **안심 중심** | 문제를 "고객이 체감하는 불안"으로 정의 | 행동 목적을 "안심 제공"으로 연결 | 결과를 "고객 안심"으로 귀결 |
| **존중 중심** | 갈등을 "틀림"이 아닌 "다양성"으로 재정의 | 행동을 "경청 → 객관적 분석 → 합의"로 설명 | 결과를 "존중과 협력으로 달성한 성과"로 귀속 |
| **균형 중심** | 문제를 "양자택일"이 아닌 "양립 과제"로 정의 | 행동을 "둘 다 추구"로 설명 | 결과를 "안정성+혁신 동시 달성"으로 귀결 |

### 3.2 재사용 가능한 스토리텔링 템플릿

#### [안정성 + 신뢰 중심 기업]
```
① [문제 설정]
   "~하는 상황이 발생했고, 이는 [시스템 안정성/고객 신뢰/조직 운영]에
   영향을 줄 수 있는 문제였다"

② [판단 기준]
   "단순히 ~하는 것이 아니라, [정량적 분석/구조적 접근/근본 원인 추적]을
   통해 해결해야 한다고 판단했다"

③ [행동 서술]
   "[구체적 행동]을 수행했고, 이는 [같은 문제 반복 방지/잠재적 리스크 사전 대비/
   시스템 안정성 확보]를 위한 것이었다"

④ [결과 해석]
   "이를 통해 [시스템 신뢰성/조직 효율성/고객 안심]의 기반을 마련할 수 있었다"
```

#### [성장 + 협업 중심 기업]
```
① [문제 설정]
   "기존에는 ~하는 방식에 안주하고 있었다 / ~라는 한계가 있었다"

② [판단 기준]
   "단순히 ~하는 것이 아니라, [더 나은/안정적인 방법]을
   찾아야 한다고 판단했다"

③ [행동 서술]
   "[팀원과 함께] [구체적 행동]을 수행했고,
   [서로 다른 관점을 공유하며] 해결책을 구체화했다"

④ [결과 해석]
   "이를 통해 [협업의 가치/팀 성장/안정성 확보]를 경험했고,
   혼자였다면 불가능했을 성과를 함께 만들어낼 수 있었다"
```

#### [본질 파악 + 신뢰 중심 기업]
```
① [문제 설정]
   "~라는 문제가 발생했고, 문제의 본질이 [표면적 증상]이 아니라
   [구조적 원인]에 있음을 파악했다"

② [판단 기준]
   "단순히 [표면적 해결]하는 것이 아니라, [구조/흐름/맥락]을
   이해하고 [본질적 해결]을 해야 한다고 판단했다"

③ [행동 서술]
   "[구조적 해결책]을 도입하여 [본질적 원인]을 해결했고,
   이를 위해 [사전 학습/선제적 준비]를 수행했다"

④ [결과 해석]
   "이를 통해 [신뢰성/안정성]을 확보했고,
   [시스템 전체/인프라]의 [가용성/안정성]을 향상시킬 수 있었다"
```

#### [공동체 + 안심 중심 기업]
```
① [문제 설정]
   "~라는 상황이 있었고, 이를 [남의 일이 아닌 공동체/고객 전체의 문제]로
   인식했다"

② [판단 기준]
   "[의견 차이/갈등]을 틀림이 아닌 다양성으로 보았고,
   [고객/사용자가 체감하는 가치] 관점에서 판단했다"

③ [행동 서술]
   "[전례 없는/과감한 시도]를 [공동체/고객을 위해] 주도적으로 추진했고,
   [존중과 협력]을 바탕으로 합의를 이끌어냈다"

④ [결과 해석]
   "이를 통해 [공동체/고객]에 [안심/긍정적 영향]을 제공할 수 있었고,
   혼자였다면 불가능했을 성과를 [협력으로] 달성했다"
```

## 4. 공통 합격 패턴

### 4.1 모든 S급 자소서에서 발견된 패턴

| 패턴 | 설명 |
|------|------|
| **두괄식 + 소제목** | 모든 문항이 핵심 메시지로 시작 |
| **Problem-Driven 전개** | 문제 상황 → 분석 → 행동 → 결과 |
| **기술의 운영 관점 서술** | 기술을 "책임/안정성" 관점으로 설명 |
| **마무리에서 회사 가치 연결** | 반드시 회사명 + 핵심 가치 언급 |
| **감정 서사의 제한적 사용** | 감정은 도입부/마무리에서만 간결하게 |
| **구체적 행동 중심** | "저는 ~했습니다"로 명확한 역할 제시 |

### 4.2 절대 발견되지 않는 패턴

| 패턴 |
|------|
| 자화자찬 ("저는 뛰어난 사람입니다") |
| 기술 스택 나열 |
| 회사 정보 복붙 |
| 막연한 포부 |
| 감정 과잉 |
| 본인 역할 불분명 |

## 5. 가치 체감 지점 (독자가 기업 가치를 '느끼게 되는' 순간)

| 단계 | 가치 체감 지점 |
|------|---------------|
| **문제 정의** | "이것은 나만의 문제가 아니라 시스템/조직/고객 전체의 문제"라고 선언할 때 |
| **판단 기준** | "다른 선택지가 있었지만, 나는 [가치 기준]에 따라 이것을 선택했다"로 대비시킬 때 |
| **행동 서술** | 행동의 목적이 "빨리 끝내기"가 아니라 "같은 문제 반복 방지"일 때 |
| **결과 해석** | 결과를 "내가 이뤘다"가 아니라 "이로 인해 ~가 가능해졌다"로 표현할 때 |
