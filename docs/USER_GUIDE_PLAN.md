# 자소서 머신 (Jasoseo Machine) 사용자 가이드 제작 플랜

이 문서는 사용자가 "자소서 머신" 앱을 처음 설치하고, 프로필을 설정한 뒤, 자기소개서를 생성하고, 크롬 확장 프로그램을 통해 채용 사이트에 자동 제출하기까지의 전체 과정을 안내하는 매뉴얼(가이드)을 작성하기 위한 **설계도 및 진행 기록**입니다.

## 🎯 가이드의 목표
1. **완전 초보자 대상:** 앱이나 CLI에 대한 지식이 없는 사용자도 스크린샷과 단계별 지침을 보고 끝까지 따라 할 수 있어야 함.
2. **명확한 흐름 (A to Z):** `환경 설정 -> 앱 설치 -> 프로필 및 에피소드 구축 -> 자소서 생성 마법사 -> 크롬 확장 연동 및 폼 자동 채우기 -> 제출`까지 누락 없는 플로우 구성.
3. **시각적 자료 강조:** 어떤 화면(스크린샷)이 필요하고 화면의 어느 부분을 강조(빨간 박스 등)해야 하는지 명시.

## 📅 단계별 가이드 구성 계획 (Draft)

### Part 1: 시작하기 전에 (필수 환경 설정)
*   **목표:** AI 엔진(Gemini, Claude) 구동을 위한 CLI 환경 구축.
*   **내용:**
    *   자소서 머신의 핵심 작동 원리 간략 소개 (왜 외부 AI CLI가 필요한가?)
    *   Node.js 설치 안내 (CLI 구동 필수 환경)
    *   [Gemini CLI 공식 설치 및 API 키 발급 가이드] 링크 및 안내
    *   [Claude CLI 공식 설치 및 API 키 발급 가이드] 링크 및 안내
    *   명령어 프롬프트/터미널에서 설치 확인 방법 (`gemini --version`, `claude --version`)
*   **필요 이미지:** 터미널에서 버전을 확인하는 캡처 화면.

### Part 2: 프로그램 설치 및 설정 (앱 & 크롬 확장)
*   **목표:** 배포된 패키지를 실행하고 웹사이트 연동을 위한 크롬 확장을 설치.
*   **내용:**
    *   **본 앱 실행:** 다운로드 받은 패키지 압축 해제 및 `jasoseo-machine.exe`(또는 실행 파일) 실행.
    *   **크롬 확장 프로그램 설치:**
        *   압축 해제된 폴더 내의 `extension` 폴더 확인.
        *   Chrome 브라우저 설정 -> 확장 프로그램 -> `개발자 모드` 켜기.
        *   `압축해제된 확장 프로그램을 로드합니다` 클릭 후 `extension` 폴더 선택.
*   **필요 이미지:**
    *   크롬 확장 프로그램 페이지에서 '개발자 모드' 토글 켜진 화면.
    *   '자소서 머신 브릿지' 확장이 크롬에 로드된 화면.

### Part 3: 나의 무기 만들기 (온보딩 및 에피소드 라이브러리)
*   **목표:** AI가 자소서를 쓸 수 있도록 내 경험(프로필, 에피소드)을 시스템에 주입.
*   **내용:**
    *   **앱 초기 설정:** 앱 실행 후 설정(톱니바퀴)에서 Gemini/Claude 경로 및 활성화 여부 확인. (Test Connection 기능 설명)
    *   **매직 온보딩 (Magic Onboarding):**
        *   기존 이력서(PDF, Word) 업로드하여 자동 프로필 추출 기능 시연.
    *   **에피소드 마이닝 (Episode Interviewer):**
        *   AI와 채팅하며 숨겨진 경험을 S-P-A-A-R-L 구조의 에피소드로 등록하는 과정.
        *   (강조) "이 에피소드들이 곧 자소서의 원천 데이터가 됩니다."
*   **필요 이미지:**
    *   앱 설정 화면 (연결 성공 초록색 불빛 캡처).
    *   이력서 파일 업로드 창 및 자동 추출 완료 알림.
    *   에피소드 마법사에서 AI와 인터뷰하며 에피소드가 생성되는 UI (스토리 카드 형태 강조).

### Part 4: 자소서 작성 마법사 (Wizard 실전)
*   **목표:** 특정 기업의 공고와 문항을 넣고 맞춤형 자소서를 생성.
*   **내용:**
    *   새 지원서 작성 클릭 -> 회사명, 직무, 채용 공고(복붙), 문항(글자수 제한 포함) 입력.
    *   **Step 0 (기업 분석):** AI가 채용 공고를 분석하여 핵심 HR 의도 2가지를 뽑아내는 화면 설명. (이후 추측성 분석 대기)
    *   **Step 1 (질문 재해석):** 문항의 의도를 기업 가치에 맞게 재정의하는 과정.
    *   **Step 2 (에피소드 매칭):** AI가 추천한 에피소드 중 내가 원하는 것을 **'승인(Approve)'**하는 과정 (핵심 보안/할루시네이션 방지 단계임 강조).
    *   **Step 3~5 (생성 스트리밍):** AI가 글을 실시간으로 써내려가는 화면. (S급 품질 소제목, S-P-A-A-R-L 구조 적용 안내)
    *   **Step 6~8 (검증):** 이중 코딩(내 이야기+회사 가치) 및 탈락 패턴 검증 단계 설명.
    *   **각 문항별 모델 교체 팁:** 막혔을 때 Model Picker로 잼나이/클로드 바꿔가며 시도하는 법.
*   **필요 이미지:**
    *   마법사 9-Step 좌측 네비게이션 트리 화면 (진행 상황 시각화).
    *   Step 2 에피소드 매칭 단계에서 '승인' 버튼에 빨간색 강조 박스.
    *   실시간으로 자소서 텍스트가 채워지는(스트리밍) 역동적인 화면.

### Part 5: 사이트 연동 및 자동 제출 (마무리)
*   **목표:** 앱에서 완성된 자소서를 크롬을 통해 채용 사이트에 클릭 한 번으로 옮기기.
*   **내용:**
    *   모든 문항 작성이 끝나고 **'전체 검토 화면(FullReview)'**으로 이동.
    *   내용 최종 확인 및 미세 수정 (Surgical Edit).
    *   Chrome 브라우저에서 실제 채용 사이트(예: 사람인, 잡코리아, 기업 자체 폼 등)를 띄워둠.
    *   자소서 머신 앱의 **'채용 사이트로 전송'** 버튼 클릭.
    *   브릿지 서버를 통해 크롬 확장 프로그램이 폼을 감지하고 자동으로 텍스트를 입력하는 마법 같은 과정 시연.
*   **필요 이미지:**
    *   앱의 FullReview 화면과 우측 상단 '채용 사이트로 전송' 버튼 캡처.
    *   실제 웹 브라우저 폼에 텍스트가 촤르륵 입력되어 있는 결과 화면.
