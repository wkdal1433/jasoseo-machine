## 1. 데이터 무결성: ID 기반 폴더 격리 (Solid Foundations)
- **목표**: 프로필 이름 변경이나 특수문자 입력에도 에피소드 데이터가 유실되지 않도록 보장.
- **액션**: 
  - `db.ts`의 `getUserProfile` 및 `saveUserProfile` 로직에서 폴더 참조 기준을 `profile.personal.name`에서 `profile.id`로 변경.
  - 기존 `episodes/[Name]` 폴더를 `episodes/[ID]`로 자동 마이그레이션하는 유틸리티 작성.

## 2. 팩트 가드: 코드 기반 2차 검증 시스템 (Post-AI Validation)
- **목표**: AI의 환각(Hallucination)을 기술적으로 차단.
- **액션**:
  - `automation/fact-checker.ts` 모듈 신설.
  - AI가 추출한 `MatchingResult`나 `Episode`의 핵심 키워드(숫자, 기술명, 고유명사)가 원문(Raw Text)에 실제로 존재하는지 `Regex/String Match`로 검증.
  - 불일치 시 해당 항목에 `[Warning: Unverified]` 플래그 부여 및 강제 `Draft` 처리.

## 3. 심리스 UX: 인라인 인터뷰 시스템 (In-place Refinement)
- **목표**: 흐름의 단절 없는 에피소드 완성 경험 제공.
- **액션**:
  - `MagicOnboarding.tsx`의 결과 화면에서 에피소드 카드 클릭 시 `EpisodeDiscoveryWizard`의 인터뷰 로직을 직접 호출하도록 통합.
  - 인터뷰 완료 시 현재 온보딩 결과 상태(색상)가 즉시 반영되는 반응형 UI 구축.

## 4. 사용자 신뢰: 안심 주입(Safe-Paste) 가이드
- **목표**: 브라우저 보안 경고로 인한 유저의 심리적 장벽 제거.
- **액션**:
  - `FullReview.tsx`의 스크립트 복사 영역 옆에 '보안 안내' 툴팁 및 팝업 추가.
  - "이 코드는 로컬에서만 생성되며, 개인정보를 외부로 전송하지 않습니다"라는 메시지와 함께 콘솔 사용법을 시각적으로 안내.
